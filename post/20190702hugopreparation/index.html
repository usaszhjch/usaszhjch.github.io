1. 下载程序

下载地址hugo。

1
2

$ hugo version
Hugo Static Site Generator v0.80.0/extended windows/amd64 BuildDate: unknown

Tips：创建好站点以后把hugo.exe放在新站点下面（例如：G:\HugoP\Excellence\hugo.exe），则可以不用设置环境变量也可以较为灵活地操作。

2. 创建站点

1

$ hugo new site Excellence

3. 安装主题

去官方网站下载主题，然后把下载的主题包放在本地博客站点根目录下里的themes文件夹，例如使用Even：

1

C:\hugo\xxx.github.io\themes\even

然后需要设置本地博客站点根目录下config.toml，需要结合具体的主题包设置对应的信息，可参考\themes\even\exampleSite\config.toml。

Tips：对于logo.jpg文件的本地路径为：

1

C:\hugo\xxx.github.io\static\images

4. 写文章

1

$ hugo new post/20190702HugoPreparation.md

注意：

  1. 写完以后保存文本类型要存为UTF-8编码而不是ANSI，否则浏览器打开页面会显示乱码，因为浏览器可能默认会用GB2312编码，而不是UTF-8编码；

  2. 文件名不能有空格。

5. 本地预览

或者是：

$ hugo server -D

如果一切顺利，打开 http://localhost:1313 即可大功告成。

7. 高级操作

7.1 新建文章模版定制

例如每次通过hugo new post/XXX.md生成的模版文章时，自动加入标签(tag)和类别(category)，需要先在C:\hugo\xxx.github.io\config.toml里定义以下内容：

1
2
3

[taxonomies]
   tag = "tags"
   category = "categories"

然后在模版文件C:\hugo\xxx.github.io\archetypes\default.md里修改。

修改前：

1
2
3
4
5

---
title: "{{ replace .TranslationBaseName "-" " " | title }}"
date: {{ .Date }}
draft: true
---

修改后：

1
2
3
4
5
6
7

---
title: "{{ replace .TranslationBaseName "-" " " | title }}"
date: {{ .Date }}
draft: true
tags: [ "", "" ]
categories: [ "", "" ]
---

根据标签或者分类访问的网址路径为：

http://localhost:1313/tags

http://localhost:1313/categories

7.2 修改网页背景

每个主题都有不同的配置文件，以hugo-zen为例，可以先在下载页面（Github）搜索background，然后在相应的css文件里尝试分析，最终找到是..\themes\hugo-zen\static\css\custom.css里。

1
2
3
4

body {
  font-family: "微软雅黑", "Roboto", "HelveticaNeue", "Helvetica Neue", Helvetica, Arial, sans-serif;
  background-color: #FCFDF8;
}

类似，修改字体和字号也在这里可以完成。

1
2
3
4

h1 { 
	font-family: "微软雅黑", "Roboto", "HelveticaNeue", "Helvetica Neue", Helvetica, Arial, sans-serif;
	font-size: 2.1rem; 
}

7.3 文章里的路径引用

Hugo默认会去根目录下的static文件夹里读取，

例如有一个jpeg文件路径是：C:\hugo\xxx.github.io\static\images\20190301233823.jpg

则在根目录下的posts文件夹里的文章提到这个路径，可以写成：/images/20190301233823.jpg。

注意路径用的斜杠不要弄错。

7.4 cname

If you’d like to use a custom domain for your GitHub Pages site, create a file static/CNAME.

Your custom domain name should be the only contents inside CNAME.

Since it’s inside static, the published site will contain the CNAME file at the root of the published site, which is a requirements of GitHub Pages.

9. GitHub 更新

  1. 查看用户名，邮箱
1
2

git config user.name
git config user.email
  1. 修改，用户名，邮箱
1
2

git config --global user.name "xxx"
git config --global user.email "xxx"
  1. 移除仓库，重新添加
1
2

git remote rm origin
git remote add origin https://github.com/XXX
  1. 解除SSL认证
1

git config --global http.sslVerify "false"
  1. 更新 DNS 缓存

cmd 窗口输入

1

ipconfig /flushdns
  1. 文件过大，超过上限

修改为 500MB，在 Git Bash 中输入以下命令：

1

git config http.postBuffer 5242880003
  1. 如果提示Filename too long，说明文件名或路径太长
1

git config --global core.longpaths true
  1. 完美解决remote: Support for password authentication was removed on August 13, 2021.

// <REPO>：包括<>在内的全部字符替换成你要访问的仓库名称

git push origin main
